## 安装命令

### 命令行工具

```bash
# Claude Code（推荐）
curl -fsSL https://claude.ai/install.sh | bash

# OpenCode（开源替代）⭐
curl -fsSL https://opencode.ai/install | bash

# Aider（开源、多模型）
pip install aider-chat

# Codex CLI
npm install -g @openai/codex

# Copilot CLI（Public Preview，需 Node.js 22+）
npm install -g @github/copilot

# Oh My OpenCode（OpenCode 增强）
bunx oh-my-opencode install

# 验证安装
claude --version
opencode --version
aider --version
```

### 认证配置

```bash
# Claude Code（OAuth 登录，不需要 API Key）
claude  # 首次运行引导登录

# OpenCode（多模型）
export OPENAI_API_KEY="sk-xxx"      # 或
export ANTHROPIC_API_KEY="sk-ant-xxx"

# Aider（支持多模型）
aider --model claude-3-5-sonnet
aider --model gpt-4o
aider --model ollama/llama3  # 本地模型

# Codex CLI
export OPENAI_API_KEY="sk-xxx"
# 或使用 ChatGPT 订阅登录
codex  # 首次运行自动引导
```

## MCP 常用命令

```bash
# 列出服务器
claude mcp list

# 添加服务器
claude mcp add [name] --transport [type] -- [command]

# 示例：添加 GitHub
claude mcp add github -- npx @modelcontextprotocol/server-github

# 移除服务器
claude mcp remove [name]

# 在 Claude Code 中查看状态
/mcp

# 调试模式
claude --mcp-debug
```

## 提示词速查

### 好的提示词

```markdown
✅ 具体明确
✅ 提供上下文
✅ 给出示例
✅ 分步骤
✅ 说明约束
```

### 避免的提示词

```markdown
❌ 太模糊："做个好看的页面"
❌ 一次太多：10个功能一起要
❌ 没上下文：不说技术栈
❌ 不给示例：纯文字描述
```

### 模板

```markdown
## 任务
[简要描述]

## 技术栈
- 框架：
- 样式：

## 需求
1.
2.
3.

## 约束
-
```

## 工具选择

```
零基础入门 → Lovable / v0.dev
        ↓
日常开发 → Cursor
        ↓
复杂项目 → Claude Code + Skills
        ↓
开源方案 → OpenCode / Aider
```

| 场景 | 推荐工具 |
|:---|:---|
| 快速原型 | Lovable |
| UI 组件 | v0.dev |
| 全栈 Demo | Bolt.new |
| 日常开发 | Cursor |
| 复杂项目 | Claude Code |
| ==开源免费== | OpenCode / Aider |
| ==多模型切换== | OpenCode / Aider |

## 安全检查

### 必须检查

- [ ] 不包含敏感信息
- [ ] 使用环境变量
- [ ] 用户输入已验证
- [ ] SQL 使用参数化
- [ ] 无 XSS 漏洞

### 风险等级

| 🟢 低风险 | 🟡 中风险 | 🔴 高风险 |
|:---|:---|:---|
| 原型 Demo | 内部工具 | 用户数据 |
| 个人项目 | 非核心功能 | 认证支付 |
